# Roadmap

Purpose

Keep a concise view of the remaining major work without preserving a large issue-by-issue backlog in the live docs.

Read this when

- You need to know what is still planned.
- You are deciding whether a change is new scope or already expected roadmap work.

Current state

Core wardrobe, recommendation, account, and planning flows are already implemented. Remaining work is mostly refinement, intelligence, and expansion.

Backlog themes

## Recommendation quality

- Build a learning pipeline from recommendation logs
- Turn garment image analysis into a complete auto-tagging flow
- Surface default tag suggestions in add/edit garment forms

## Reliability and UX

- Complete the accessibility follow-up
- Expand E2E coverage around core user flows
- Improve operational monitoring, rate limiting, and structured logging

## Account and security

- Add connected sessions and optional 2FA
- Strengthen server-side protections on sensitive API routes where needed

## Product expansion

- Recommendation digest or re-engagement workflow
- Optional product feed sync and provider operations
- Social sharing/community features
- Mobile app exploration

Planning rule

- Keep this file high-level.
- Detailed specs, audits, and historical issue breakdowns belong in `docs/archive/`.
