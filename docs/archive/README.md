# Archive

Purpose

Mark the contents of this folder as historical material that is preserved for reference but not treated as current developer guidance.

Read this when

- You need old planning context, audits, issue specs, or presentation artifacts.
- A live doc links here explicitly for historical background.

Current state

Everything under `docs/archive/` is non-canonical. The active docs live directly under `docs/`, `docs/features/`, and `docs/reference/`.

Archive categories

- `audits/`: reviews, audits, and diagnostic writeups
- `implementation/`: phased implementation notes and summaries
- `issues/`: detailed issue-by-issue backlog specs
- `legacy/`: superseded docs merged into the live set
- `product/`: older product-planning documents replaced by concise live docs
- `presentations/`: presentation assets and related supporting files
